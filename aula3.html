AULA 3: CSS avançado

Com o avanço do html e css, foi possivel fazer coisas que antes eram impossiveis, que antes eram feitas com imagem ou até com o uso de javascript, por isso, veremos algumas das tecnicas mais avançadas de css

Seletores avançados

Vimos seletores mais avançdos como o de atributo, mas agora veremos outros

Seletores irmãos

<article>
    <h1>Titulo</h1>
    <p>Inicio</p>


    <h2>Subtitulo</h2>
    <p>Texto</p>
    <p>Mais texto</p>
</article>

Se quisermos estiliza todos os paragrafos apos o subtitulo?

Usamos o seletor de irmão (siblings) (~).

Ele serve para selecionar elementos que são irmãos de um elemento específico.

Ficando:

h2~p{
    color: red;
}

Seletores irmãos adjacente

O seletor de irmãos adjacente (+) seleciona o elemento imediatamente seguinte ao elemento especificado.

Ficando:

h2+p{
    color: red;
}

Assim apenas o "texto" é aplicado a cor red, o "mais texto" não será afetado.

Seletor de filho direto

O seletor de filho direto (>) serve para selecionar elementos que estão imediatamente dentro de outro, sem considerar os "netos" ou elementos em níveis mais profundos da estrutura.

Imagine o seguinte cenário em HTML:

<article>
    <h1>Título principal</h1>

    <section>
        <h1>Título da seção</h1>
    </section>
</article>

Se você quiser alterar a cor apenas do título principal, usar um seletor de hierarquia comum (com espaço) não vai funcionar, pois ele pintaria ambos os títulos de azul:

/* Errado: isso vai selecionar TODOS os h1 dentro do article */
article h1 {
    color: blue;
}

Para resolver isso, usamos o operador maior que (>). Ele garante que apenas o h1 que é filho imediato do article receba o estilo:

/* Correto: seleciona apenas o h1 que é filho direto */
article > h1 {
    color: blue;
}

Seletor de negação

O seletor de negação (:not) serve para aplicar estilos a elementos que não possuem uma determinada característica, como uma classe ou um ID específico.

Considere esta lista de parágrafos:

<p>Texto</p>
<p>Outro texto</p>
<p class="especial">Texto especial</p>
<p>Mais texto</p>

Se a intenção for deixar todos os parágrafos cinzas, exceto aquele que foi marcado com a classe "especial", usamos a pseudo-classe :not() passando o alvo da exceção entre os parênteses:

/* Seleciona todos os parágrafos que NÃO têm a classe .especial */
p:not(.especial) {
    color: gray;
}

Pseudo-classes

Imagine que temos uma lista simples no HTML e queremos estilizar elementos específicos dela — por exemplo, deixar o primeiro item vermelho e o último azul:

<ul>
    <li>Primeiro item</li>
    <li>Segundo item</li>
    <li>Terceiro item</li>
    <li>Quarto item</li>
</ul>

Uma solução comum seria adicionar classes manuais no HTML. Porém, isso exige alteração estrutural e manutenção constante se a ordem dos itens mudar. Para resolver isso de forma inteligente, o CSS oferece as pseudo-classes.

Elas funcionam como classes pré-definidas pelo próprio navegador, permitindo selecionar elementos com base em sua posição ou estado sem mexer no HTML. Para diferenciá-las de uma classe comum, usamos o caractere de dois pontos (:) em vez do ponto.

/* Seleciona o primeiro li filho */
li:first-child {
    color: red;
}

/* Seleciona o último li filho */
li:last-child {
    color: blue;
}

O seletor nth-child

Para um controle ainda mais flexível, utilizamos o seletor :nth-child(), que recebe um índice ou uma expressão matemática entre os parênteses para definir quais elementos serão afetados.

Podemos selecionar um elemento exato informando o seu número:

/* Seleciona especificamente o terceiro item */
li:nth-child(3) {
    color: yellow;
}

Também é possível criar um padrão alternado (como uma lista zebrada) utilizando expressões aritméticas para atingir índices pares e ímpares:

/* Seleciona os elementos pares (2, 4, 6...) */
li:nth-child(2n) {
    color: green;
}

/* Seleciona os elementos ímpares (1, 3, 5...) */
li:nth-child(2n+1) {
    color: blue;
}

Pseudo-classes de estado

Outra utilidade importante das pseudo-classes é reagir às ações do usuário em tempo real, capturando o estado atual de um elemento (como passar o mouse por cima ou clicar) de forma nativa, sem depender de scripts externos.

A pseudo-classe :hover é acionada no exato momento em que o ponteiro do mouse é posicionado sobre o elemento:

/* Altera o fundo quando o usuário passa o mouse pelo link */
a:hover {
    background-color: #FF00FF;
}

Especificamente para links (âncoras), existem outras pseudo-classes fundamentais para gerenciar o comportamento visual de navegação:

/* Seleciona links padrão que possuem o atributo href */
a:link {
    background-color: #FF0000;
}

/* Seleciona links que o usuário já visitou no histórico */
a:visited {
    background-color: #00FF00;
}

/* Seleciona o link durante o clique ativo do mouse */
a:active {
    background-color: #0000FF;
}

Pseudo-elementos

Enquanto as pseudo-classes selecionam elementos que já existem com base em sua posição ou estado, os pseudo-elementos permitem estilizar partes específicas do conteúdo de um elemento que sequer possuem uma tag própria no HTML (ou até mesmo gerar novos conteúdos visuais).

Para diferenciar visualmente os pseudo-elementos das pseudo-classes no CSS moderno, utilizamos a sintaxe de dois pontos duplos (::).

Imagine que você queira criar um efeito de revista, aumentando o tamanho apenas da primeira letra de um parágrafo comum:

<p>A Caelum tem os melhores cursos!</p>

Em vez de poluir o HTML envolvendo a letra "A" com uma tag desnecessária, usamos o pseudo-elemento ::first-letter diretamente no CSS:

/* Seleciona e estiliza apenas a primeira letra do parágrafo */
p::first-letter {
    font-size: 200%;
}

Da mesma forma, se precisarmos aplicar um estilo exclusivo para a primeira linha de um bloco de texto (independentemente do tamanho da tela do usuário), podemos usar o ::first-line:

/* Deixa apenas a primeira linha do parágrafo em itálico */
p::first-line {
    font-style: italic;
}

Inserindo novos conteúdos via CSS

Existem dois pseudo-elementos muito poderosos chamados ::before e ::after. Eles servem para injetar conteúdo puramente estético antes ou depois do conteúdo real de um elemento, mantendo o HTML limpo e focado apenas no significado do texto (sem afetar leitores de tela ou mecanismos de busca).

Considere esta lista simples de links:

<a href="#">Link 1</a>
<a href="#">Link 2</a>

Se quisermos cercar esses links com colchetes apenas por uma escolha de design, usamos a propriedade content dentro dos pseudo-elementos:

/* Adiciona um colchete antes do texto do link */
a::before {
    content: '[ ';
}

/* Adiciona um colchete depois do texto do link */
a::after {
    content: ' ]';
}

Podemos ir além e combinar pseudo-elementos com seletores de atributo. Por exemplo, para identificar de forma automática e visual todos os links externos da página que começam com "http://", adicionando o aviso "(externo)" após o texto deles:

/* Identifica links externos e insere um aviso ao final */
a[href^="http://"]::after {
    content: ' (externo)';
}

Bordas arredondadas com border-radius

Antes do CSS3, a única forma de criar cantos arredondados em um elemento era utilizando imagens de fundo, o que aumentava o tempo de carregamento da página e dificultava muito a manutenção do código.

Com a propriedade border-radius, podemos definir o raio de arredondamento dos cantos de um elemento de forma totalmente nativa e simples:

/* Aplica um raio de 5px em todos os quatro cantos uniformemente */
div {
    border-radius: 5px;
}

Customizando cantos específicos

A propriedade border-radius é uma propriedade abreviada (shorthand). Isso significa que podemos passar de um a quatro valores para controlar o arredondamento de cada canto individualmente, seguindo a ordem horária a partir do canto superior esquerdo:

/* 1 Valor: Aplica o mesmo raio (15px) em todos os cantos */
.a {
    border-radius: 15px;
}

/* 2 Valores: [Superior-Esquerdo e Inferior-Direito] [Superior-Direito e Inferior-Esquerdo] */
.b {
    border-radius: 5px 20px;
}

/* 3 Valores: [Superior-Esquerdo] [Superior-Direito e Inferior-Esquerdo] [Inferior-Direito] */
.c {
    border-radius: 5px 20px 50px;
}

/* 4 Valores: [Superior-Esquerdo] [Superior-Direito] [Inferior-Direito] [Inferior-Esquerdo] */
.d {
    border-radius: 5px 20px 50px 100px;
}
h2>Sombras em textos com text-shadow

A propriedade text-shadow do CSS3 permite adicionar sombras diretamente em textos de forma nativa. Sua sintaxe básica recebe os valores de deslocamento horizontal, deslocamento vertical e a cor da sombra:

/* Sombra deslocada 10px para a direita e 10px para baixo, na cor vermelha */
p {
    text-shadow: 10px 10px red;
}

Adicionando efeito de desfoque (blur)

Também podemos passar um terceiro valor numérico antes da cor para definir o nível de espalhamento ou desfoque (blur) da sombra, deixando o efeito mais suave:

/* Deslocamento horizontal, vertical, raio de desfoque (5px) e cor */
p {
    text-shadow: 10px 10px 5px red;
}

Múltiplas sombras

Se quisermos criar efeitos visuais mais complexos, o CSS permite aplicar múltiplas sombras ao mesmo elemento. Para isso, basta separar cada bloco de configuração por uma vírgula:

/* Aplica duas sombras com direções e desfoques diferentes ao mesmo tempo */
p {
    text-shadow: 10px 10px 5px red, -5px -5px 4px red;
}

Sombras em elementos com box-shadow

Além de aplicar sombras em textos, o CSS3 permite adicionar sombras em qualquer elemento de bloco (como divs, imagens ou parágrafos) através da propriedade box-shadow.

A sua sintaxe básica segue a mesma lógica do text-shadow, recebendo os valores de deslocamento horizontal, deslocamento vertical e a cor:

/* Sombra deslocada 20px para a direita e 20px para baixo, na cor preta */
div {
    box-shadow: 20px 20px black;
}

Podemos adicionar um terceiro valor numérico para controlar o nível de desfoque (blur) da sombra:

/* Deslocamento horizontal, vertical, raio de desfoque (20px) e cor */
div {
    box-shadow: 20px 20px 20px black;
}

Diferente do text-shadow, o box-shadow aceita um quarto valor numérico opcional (antes da cor) chamado spread. Ele serve para expandir ou contrair o tamanho da sombra em todas as direções:

/* Deslocamento, desfoque (20px), expansão (30px) e cor */
div {
    box-shadow: 20px 20px 20px 30px black;
}

Por fim, podemos adicionar a palavra-chave inset no início da propriedade. Isso faz com que a sombra seja projetada para o lado de dentro do elemento, criando um efeito de profundidade interna:

/* Sombra interna centralizada com 10px de desfoque */
div {
    box-shadow: inset 0 0 10px black;
}

Opacidade e cores com RGBA

A propriedade opacity serve para alterar a opacidade completa de um elemento, aceitando valores decimais que variam entre 0 (totalmente transparente) e 1 (totalmente opaco):

/* Deixa todo o parágrafo (fundo e texto) com 30% de opacidade */
p {
    opacity: 0.3;
}

O problema do opacity é que ele afeta todos os elementos filhos. Se você aplicar opacidade em uma caixa de texto, as letras de dentro também ficarão transparentes e difíceis de ler.

Para resolver isso e aplicar transparência apenas na cor de fundo (mantendo o texto totalmente visível e nítido), o CSS3 introduziu a função rgba(). Além dos três canais normais de cores (Red, Green, Blue de 0 a 255), ela recebe um quarto argumento: o canal Alpha (de 0 a 1):

/* Formas equivalentes de declarar a cor branca */
p {
    color: white;
    color: #FFFFFF;
    color: rgb(255, 255, 255);
}

/* Aplicando o canal Alpha: branco com 80% de opacidade */
p {
    color: rgba(255, 255, 255, 0.8);
}

Com essa técnica, conseguimos criar um fundo preto translúcido atrás de um texto sem deixar o texto em si transparente:

/* Fundo preto com 30% de opacidade, mas texto 100% branco e nítido */
p {
    background-color: rgba(0, 0, 0, 0.3);
    color: white;
}

Prefixos dos fabricantes

Muitos recursos do HTML5 e do CSS3 passam por um longo período experimental antes de se tornarem normas oficiais. Para permitir que os desenvolvedores testem essas novidades com antecedência, os navegadores costumam implementá-las de forma provisória adicionando um prefixo específico antes do nome da propriedade.

Os prefixos dos fabricantes mais conhecidos são:

  • -webkit-: Navegadores baseados em WebKit (Chrome, Safari, iOS, Android)
  • -moz-: Firefox (Mozilla)
  • -ms-: Internet Explorer / Edge antigo (Microsoft)
  • -o-: Opera

Atualmente, propriedades consagradas como border-radius e box-shadow são interpretadas nativamente por qualquer navegador moderno sem necessidade de prefixos. No entanto, se o objetivo do projeto for garantir a compatibilidade com sistemas legados ou versões muito antigas, a boa prática exige empilhar as regras deixando a oficial por último:

/* Chrome antigo (até v3) e Safari antigo (até v4) */
p {
    -webkit-border-radius: 5px;
}

/* Firefox antigo (até v3.6) */
p {
    -moz-border-radius: 5px;
}

/* Sintaxe oficial e padrão para todos os navegadores modernos */
p {
    border-radius: 5px;
}

Gradientes com CSS3

Os gradientes do CSS3 permitem criar transições suaves entre duas ou mais cores diretamente pelo código, eliminando a dependência de imagens de fundo pesadas. Isso deixa as páginas mais leves e garante que o efeito visual se adapte perfeitamente a qualquer resolução de tela.

A especificação oficial divide os gradientes em duas categorias principais: lineares (em linha reta) e radiais (a partir de um ponto central):

/* Gradiente linear padrão (de cima para baixo) */
.linear {
    background: linear-gradient(white, blue);
}

/* Gradiente radial (do centro para as bordas) */
.radial {
    background: radial-gradient(white, blue);
}

Direções, angulações e múltiplas paradas

Podemos customizar a trajetória do gradiente indicando uma direção explícita (como to right ou to bottom) ou utilizando graus de inclinação (como 45deg). Também é possível definir múltiplos pontos de parada inserindo porcentagens para determinar onde cada cor deve atingir sua força total:

/* Gradiente linear direcionado com início no topo (top) indo para a base (bottom) */
.linear-direcionado {
    background: linear-gradient(to bottom, white, blue);
}

/* Gradiente complexo inclinado a 45 graus com três paradas distintas de cor */
.gradiente-avancado {
    background: linear-gradient(45deg, #F0F9FF 0%, #CBEBFF 47%, #A1DBFF 100%);
}

A evolução da sintaxe e retrocompatibilidade

Embora a sintaxe atual seja universal entre os navegadores de hoje, o rascunho dos gradientes mudou drasticamente ao longo do tempo. Versões antigas interpretavam os parâmetros de direção de forma invertida ou exigiam estruturas de funções bem mais complexas.

Se um projeto demandar suporte total para navegadores do passado (como o Chrome até a versão 9 ou o Safari até a versão 5), o bloco de código precisará incluir as variações prefixadas e as estruturas legadas equivalentes:

.linear {
    /* Sintaxe WebKit obsoleta e altamente complexa (Chrome v9- / Safari v5-) */
    background: -webkit-gradient(linear, left top, left bottom, color-stop(0%, white), color-stop(100%, blue));

    /* Sintaxe intermediária prefixada (Atenção: usava "top" em vez de "to bottom") */
    background: -webkit-linear-gradient(top, white, blue);
    background: -moz-linear-gradient(top, white, blue);
    background: -o-linear-gradient(top, white, blue);

    /* Padrão atual e definitivo */
    background: linear-gradient(to bottom, white, blue);
}

Geração de gradientes e estratégias de fallback

Uma prática recomendada ao trabalhar com gradientes é declarar uma cor de fundo simples e sólida imediatamente antes da regra do gradiente. Como os navegadores antigos não reconhecem a função do gradiente, eles simplesmente ignoram essa linha e renderizam a cor sólida, garantindo que o conteúdo continue legível e usável.

Por outro lado, os navegadores modernos processam a cor sólida primeiro e, em seguida, aplicam o gradiente por cima, substituindo-a. Essa estratégia de design é conhecida como melhoria progressiva (progressive enhancement):

.gradiente {
    /* Fallback para navegadores muito antigos que não suportam gradientes */
    background: #CBEBFF;

    /* Aplicado por navegadores modernos que entendem a propriedade */
    background: linear-gradient(45deg, #F0F9FF 0%, #CBEBFF 47%, #A1DBFF 100%);
}

Progressive Enhancement

Ao planejar a adoção de novas propriedades visuais, surge uma dúvida comum em relação aos navegadores antigos que não possuem suporte a essas tecnologias. O melhor caminho não é deixar de utilizar os recursos modernos, sacrificando a experiência da maioria dos usuários, mas sim adotar estratégias que garantam a usabilidade do site em qualquer cenário.

Progressive Enhancement vs. Graceful Degradation

Existem duas abordagens principais para lidar com a compatibilidade entre diferentes gerações de navegadores:

  • Graceful Degradation (Degradação Graciosa): O desenvolvimento foca primariamente nos navegadores modernos. O objetivo é fazer com que o site "degrade" de forma organizada nos sistemas antigos, ocultando ou simplificando recursos indisponíveis sem quebrar a estrutura básica.
  • Progressive Enhancement (Melhoria Progressiva): A lógica funciona em sentido inverso. Inicia-se o desenvolvimento garantindo o funcionamento e a usabilidade em qualquer ambiente básico. A partir dessa base sólida, adicionam-se camadas de melhorias visuais e interativas que serão ativadas apenas nos navegadores que as suportarem.

Com o CSS3, a melhoria progressiva permite criar páginas perfeitamente navegáveis com bordas quadradas ou fundos sólidos em sistemas legados, enquanto usuários de navegadores modernos desfrutam de cantos arredondados, gradientes e transições complexas.

Para um aprofundamento sobre o tema, consulte o artigo no blog da Caelum: http://blog.caelum.com.br/css3-e-progressive-enhancement/

CSS3 Transitions

A propriedade transition permite animar de forma suave a mudança de valores em regras CSS, evitando alterações bruscas de estado quando um gatilho (como o posicionamento do mouse) é acionado.

Considere o exemplo básico abaixo, onde a posição de um elemento é alterada instantaneamente ao passar o mouse (:hover):

#teste {
    position: relative;
    top: 10px;
}

#teste:hover {
    top: 30px;
}

Para fazer com que esse deslocamento aconteça de maneira gradual e sutil, indicando a trajetória do elemento ao longo do tempo, aplica-se a transição:

#teste:hover {
    transition: top 2s;
}

Tipos de animação (Timing Functions)

O CSS3 disponibiliza diferentes curvas de velocidade para controlar o ritmo do movimento ao longo da duração definida:

  • linear: Mantém uma velocidade constante do início ao fim.
  • ease: Inicia o movimento de forma suave, acelera no meio e desacelera no final (comportamento padrão).
  • ease-in: Começa em velocidade reduzida e acelera gradualmente até o final.
  • ease-out: Inicia rapidamente e reduz a velocidade gradualmente no final.
  • ease-in-out: Combina o início lento e o final lento, suavizando as duas extremidades.
  • cubic-bezier(x1, y1, x2, y2): Permite mapear curvas de velocidade personalizadas e complexas através de coordenadas matemáticas.
#teste:hover {
    transition: top 2s ease;
}

Para planejar e testar curvas de velocidade customizadas baseadas em Bézier, pode-se utilizar a ferramenta auxiliar disponível em: http://www.roblaplaca.com/examples/bezierBuilder/

Múltiplas propriedades e o atalho all

É possível monitorar e animar diversas propriedades simultaneamente dentro do mesmo seletor, separando as regras por vírgulas, incluindo transições de cores:

#teste {
    position: relative;
    top: 10px;
    color: white;
}

#teste:hover {
    top: 30px;
    color: red;
    transition: top 2s, color 1s ease;
}

Caso todas as propriedades modificadas devam compartilhar exatamente o mesmo tempo de duração e a mesma curva de velocidade, utiliza-se a palavra-chave all (que também atua como o valor padrão da propriedade):

#teste:hover {
    transition: all 2s ease;
}

CSS3 Transforms

A especificação de transformações permite alterar a geometria e o posicionamento visual de um elemento na tela (como rotação, escala e distorção) de forma isolada, sem causar impacto ou deslocamento na estrutura dos demais elementos ao redor.

Translate

A função translate modifica o posicionamento do elemento nos eixos bidimensionais. É possível isolar a movimentação na horizontal com translateX, na vertical com translateY, ou combinar ambos parâmetros na função genérica:

header {
    /* Move o elemento 50px para a direita no eixo horizontal */
    transform: translateX(50px);
}

#main {
    /* Move o elemento 20px para cima no eixo vertical */
    transform: translateY(-20px);
}

footer {
    /* Move o elemento simultaneamente nos eixos X (40px) e Y (-20px) */
    transform: translate(40px, -20px);
}

Rotate

A função rotate altera a inclinação angular do elemento com base em uma unidade de graus (deg). Valores positivos rotacionam no sentido horário, enquanto valores negativos rotacionam no sentido anti-horário:

#menu-departamentos {
    transform: rotate(-10deg);
}

Scale

A função scale modifica a escala de dimensionamento do elemento em relação ao seu tamanho original (onde 1 equivale a 100%). O valor 1.2, por exemplo, amplia o elemento em 20%:

#novidades li {
    /* Altera a escala proporcional total do elemento */
    transform: scale(1.2);
}

Skew

A função skew causa uma distorção ou inclinação nos cantos do elemento com base em graus, alterando sua inclinação sem modificar a altura ou largura base:

footer {
    /* Distorce as laterais do elemento no eixo horizontal */
    transform: skewX(10deg);
}

#social {
    /* Distorce as linhas superiores e inferiores no eixo vertical */
    transform: skewY(10deg);
}

Combinação de transformações

Para aplicar múltiplos efeitos de transformação sobre o mesmo seletor, deve-se listar as funções sequencialmente, separadas por espaços simples, dentro da mesma propriedade transform:

html {
    /* Rotaciona o elemento em -30 graus e reduz sua escala para 40% do tamanho */
    transform: rotate(-30deg) scale(0.4);
}

Especificidade de seletores CSS

Quando declaramos uma série de seletores CSS, é bem possível que nos deparemos com situações em que mais de um seletor esteja aplicando propriedades visuais ao mesmo elemento do HTML. Nesse caso, é necessário saber qual seletor tem precedência sobre os outros, a fim de resolver conflitos e garantir que as propriedades desejadas estejam sendo aplicadas aos elementos corretos.

Acúmulo e substituição de propriedades

O comportamento padrão dos seletores CSS, quando não há conflitos entre propriedades, é que as regras de mais de um seletor para um mesmo elemento sejam acumuladas. Veja no exemplo a seguir:

p {
    color: navy;
}

p {
    font-size: 16px;
}

No caso anterior, utilizamos o mesmo seletor duas vezes no CSS. Isso faz com que as propriedades sejam acumuladas em todos os elementos selecionados. Portanto, todos os parágrafos do documento serão exibidos na cor azul-marinho (navy) e com a fonte no tamanho de 16px.

Quando há conflito entre propriedades de seletores equivalentes, ou até mesmo dentro de um mesmo seletor, aplica-se a propriedade declarada por último no arquivo:

p {
    color: navy;
    font-size: 12px;
}

p {
    font-size: 16px;
}

Nesse cenário, há um conflito entre as propriedades font-size. Como os seletores possuem o mesmo peso, os parágrafos serão exibidos com o tamanho de 16px. A declaração anterior de 12px é sobrescrita pela nova regra declarada mais abaixo no código. A cor navy continua aplicada normalmente.

O cálculo da especificidade

Seletores equivalentes têm suas propriedades sobrescritas conforme a ordem de declaração. Mas o que acontece quando temos diferentes tipos de seletores apontando para o mesmo elemento?

Cada tipo de seletor tem um peso diferente quando o navegador os interpreta. Pensamos em uma regra simples: quanto mais específico for o seletor, maior será o seu valor de especificidade.

  • Seletor de tag (ex: div {}): É bem genérico. Aplica-se a todas as tags do documento, sem levar em consideração seus atributos. Por isso, possui valor baixo de especificidade.
  • Seletor de classe (ex: .destaque {}): É mais específico, pois nós determinamos quais elementos recebem essa classe no HTML. Seu valor é maior que o seletor de tag.
  • Seletor de ID (ex: #cabecalho {}): É altamente específico, já que só podemos ter um único elemento com determinado ID por página. Possui o maior valor entre os três.

Para calcular a prioridade de seletores compostos ou combinados, podemos somar os pontos de cada tipo em suas respectivas posições decimais (representadas pelo formato fictício 000, correspondendo a centenas para ID, dezenas para classe e unidades para tag):

p { /* Especificidade: 001 */
    color: blue;
}

.destaque { /* Especificidade: 010 */
    color: red;
}

#cabecalho { /* Especificidade: 100 */
    color: green;
}

Nos seletores combinados, basta somar os valores de cada elemento constituinte para obter o peso final do grupo:

#rodape p { /* Especificidade: 101 (1 ID + 1 Tag) */
    font-size: 11px;
}

#cabecalho .conteudo h1 { /* Especificidade: 111 (1 ID + 1 Classe + 1 Tag) */
    color: green;
}

.conteudo div p span { /* Especificidade: 013 (1 Classe + 3 Tags) */
    font-size: 13px;
}

Quanto maior o valor final da especificidade, maior será a prioridade do seletor. Dessa maneira, um seletor com peso 013 sobrescreverá com sucesso as propriedades conflitantes de um seletor com peso 001 aplicado sobre o mesmo elemento.
