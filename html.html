AULA 1: HTML e CSS

O que é HTML?

Esqueleto do navegador, sua estrutura de forma bruta, sem a estética. De forma mais formal, é a marcação do conteúdo na página WEB.

Tags

São como etiquetas. O HTML define que os conteúdos sejam etiquetados com a etiqueta (tag) que mais convém para cada conteúdo.

Dessa forma, a sintaxe do HTML é feita na forma <_>.

Assim, o que vem dentro do sinal de maior e menor deve ter um nome específico que melhor representa o conteúdo que você quer colocar, dessa forma: <TAG>.

Por exemplo:

<h1>: É uma tag para títulos principais da página. Dessa forma: <h1>TITULO_PRINCIPAL</h1>.

Note que usamos o </NOME_DA_TAG> para fechar o conteúdo.

Um detalhe interessante é que nem toda tag vai ter um conteúdo, mas falaremos disso depois; por enquanto veremos aquelas que têm. O que você precisa saber por agora é que se ela tem conteúdo precisa desse fechamento, se não tiver conteúdo não precisa ser fechada.

Atributo

Muitas tags têm atributos. Ele vem para definir algo que apenas a tag não tem como saber, então precisa de atributo.

A sintaxe é: <NOME_DA_TAG ATRIBUTO=VALOR>.

Por exemplo:

<img src="endereço_da_imagem">

Veja que não tem conteúdo em si, o atributo vai na tag, dentro do sinal de maior e menor, então não precisa de tag de fechamento (/nome_da_tag).

Estrutura válida do documento

Para ser válido, o documento precisa de:

  1. <html>
  2. <head>
  3. <body>

Dessa forma, veremos cada uma:

<html>

Na estrutura do documento, inserimos, antes de tudo, essa tag: <html>. Dentro dela temos duas tags. Quando uma tag está dentro de outra, podemos dizer que elas são filhas (internas) e mãe (externa). As duas filhas podem ser chamadas de irmãs.

<html>
    <head></head>
    <body></body>
</html>
<head>

Contém informações sobre o documento que interessam apenas ao navegador e não ao usuário visitante.

Dentro dela precisa obrigatoriamente ter um título, e usamos a tag: <title>.

Como o title está dentro do head, não vai aparecer para o usuário na tela; ela é o título do documento e ficará na barra de título no topo do navegador.

Para a configuração de idiomas como o português, que usa caracteres com acento, usamos o encoding ou charset. Fica na tag <meta>.

Atualmente usamos o UTF-8, mas já usamos o latin1.

<html>
    <head>
        <meta charset = "utf-8">
        <title>TITULO_DO_DOCUMENTO</title>
    </head>
    <body></body>
</html>
<body>

Aqui, tudo que estiver será exibido para o usuário visitante.

Deverá ter uma ou mais tags ali dentro.

<html>
    <head>
        <meta charset = "utf-8">
        <title>TITULO_DO_DOCUMENTO</title>
    </head>
    <body>
        <h1>Título_principal_que_aparece_para_o_usuário</h1>
    </body>
</html>
DOCTYPE

Não é tag, são instruções especiais.

Fala para o navegador qual versão HTML deve ser usada.

Usamos o <!DOCTYPE html>, que indica que deve ser usada a versão mais recente, que atualmente é a versão 5.

Estrutura de um documento HTML

Veremos 3 tags por agora: títulos, parágrafo e ênfase.

  1. Títulos
  2. Parágrafos
  3. Ênfase
Títulos

Usadas as tags de heading, marcadas por: <h_numero>.

<h1> até <h6>, seguindo a ordem de importância.

Respeitar o nível de importância define o tamanho da forma com que ela vai se apresentar no site, além de que vai determinar a prioridade de algumas ferramentas, como a busca do Google.

Parágrafos

Tags de parágrafos vêm marcadas como: <p>conteúdo</p>.

Ênfase

Tags para negrito = <strong>, para itálico = <em>.

IMAGENS

Tag <img> define uma imagem em uma página.

Deve ter dois atributos preenchidos: src e alt.

O atributo src é obrigatório, aponta o local da imagem. O alt é um texto alternativo para o caso de a imagem não carregar.

O HTML5 tem uma nova tag, o figure, onde você pode colocar o img e, além disso, o figcaption, onde você pode colocar uma legenda.

<figure>
    <img src = "caminho_da_imagem" alt = "texto_alternativo" >
    <figcaption>Legenda_para_essa_imagem</figcaption>
</figure>

A estrutura dos arquivos de um projeto

Aqui temos recomendações, mas não regras, podendo adaptar para seu projeto.

Geralmente fica tudo em uma pasta, e para ela tem uma "capa". Essa capa é uma página inicial que indica para o visitante quais são as outras páginas que fazem parte desse projeto e como pode acessá-las, como um índice.

Pasta_nome_do_projeto

index.html

Então para o CSS usamos uma pasta, e para o JS também, além de ser conveniente ter uma pasta para imagem.

Pasta_nome_do_projeto

index.html

sobre.html

css/

js/

img/

Note que se tiver outros HTML coloque no mesmo nível hierárquico do index.html.

O que é CSS?

Quando colocamos uma tag, e ela aparece no navegador, ela tem um estilo padrão, como tamanho, fonte e cor. Mas podemos customizar esse estilo com CSS.

CSS = linguagem separada do HTML para cuidar da estilização da página.

Antigamente usava-se no próprio HTML. Por exemplo: <h1><font color="red">Texto</font></h1>

Assim ficaria vermelho o texto. Mas não usamos mais tags de estilo. Usamos o CSS.

Sintaxe e Inclusão do CSS

Propriedade e valores separados por um sinal de dois pontos ":", e cada conjunto de propriedade+valor separa-se de outra propriedade+valor com um ponto e vírgula ";"

Por exemplo:

{
    color: blue;
    background-color: yellow;
}

Nesse exemplo as tags que forem encapsuladas com esse CSS vão ficar azuis, com fundo amarelo.

Podemos usar o CSS no HTML também. Mas você não acabou de dizer que ele existe justamente para sair do HTML? Calma, antes do CSS usávamos tags de estilo, não usamos mais tags de estilo, usamos o CSS, que tem uma sintaxe específica, e que pode ser usada no HTML ou em um arquivo separado (melhor prática).

Exemplo de CSS dentro do HTML: <p style="color: blue; background-color: yellow;"> Conteúdo </p>

Além disso, outra forma de usar CSS dentro do HTML seria, ao invés de tag de estilo (não usadas) e ao invés de integrar a uma tag como vimos ela no <p>, podemos usar uma tag criada para o CSS: <style>. Note que não é uma tag de estilo (que não deve ser usada), é uma tag própria para o CSS, que agrupa o CSS em sua própria sintaxe.

Tanto nessa última forma quanto no CSS em outro arquivo usamos o seletor CSS para indicar qual tag vamos estilizar.

<style>
    p {
        color: blue;
        background-color: yellow;
    }
</style>

No exemplo acima significa que todas as tags p (p sendo o seletor CSS) terão esse estilo.

Observação: Essa forma, com a tag style, deve ser colocada no head.

Arquivo externo

Para ter o CSS em um outro arquivo, devemos conectar o CSS ao HTML; para isso usamos a extensão .css, ou seja, seu arquivo CSS deve ser NOME_DO_ARQUIVO.css.

Colocamos um link no head: <link rel="stylesheet" href="estilo.css">

Propriedades tipográficas e fontes

Propriedade font-family

Recebe o valor com ou sem aspas, primeiramente veremos com as famílias.

As fontes têm famílias:

Serif: São chamadas de fontes serifadas, por conta do pequeno ornamento em sua terminação, e costuma ser a padrão. As mais usadas são times e times new roman.

Sans-serif: Podemos usar sem serifas, como por exemplo Arial, Helvetica.

Monospace: Como por exemplo a fonte Courier.

h1 {
    font-family: serif;
}

Podemos usar a fonte específica além da família.

h1 {
    font-family: "times", serif;
}

Podemos colocar uma segunda opção para o navegador caso ele não tenha disponível a primeira.

h1 {
    font-family: "times", "times new roman", serif;
}

Se não tiver a primeira, verificará a segunda; se não tiver a segunda, diz para ser qualquer uma serif.

Propriedade font-style

Define o estilo da fonte: normal, italic e oblique.

Alinhamento e decoração de texto

Propriedade text-align

Alinhamento do texto.

text-align: right = direita

text-align: left = esquerda

text-align: center = centro

text-align: justify = justificado

É possível configurar outros espaçamentos de texto:

p {
    line-height: 3px; /* tamanho da altura de cada linha */
    letter-spacing: 3px; /* tamanho do espaço entre cada letra */
    word-spacing: 5px;  /* tamanho do espaço entre cada palavra */
    text-indent: 30px;  /* tamanho da margem da primeira linha 
        do texto */
}

Imagem de Fundo

h1 {
    background-image: url(arquivo.jpg);
}

Serve para colocar uma imagem no fundo da tag. Precisa estar na mesma pasta CSS.

Bordas

body {
    border-color: red; /*Cor*/
    border-style: solid; /*Estilo de exibição*/
    border-width: 1px; /*Largura*/
}

Pode ser otimizado como:

body {
    border: 1px solid red;
}

Cores na web

Podemos definir cor pelo seu nome:

h1 {
    color: red;
}

Mas não tem variação só escrevendo o nome; se você quer um tom específico é mais fácil usar a composição RGB. 16 milhões de cores combinando 3 cores bases: Vermelho (Red), Verde (Green), Azul (Blue). Então cada uma dessas três vai de 0 a 255; por exemplo, amarelo forte = (255, 255, 0).

A sintaxe:

h1 {
    color: rgb(255, 255, 0);
}

No entanto, o mais usado é a notação hexadecimal. Usamos o #, depois temos dois caracteres para o Red, dois para o Green e dois para o Blue. Com dois caracteres em hexadecimal, podemos escrever facilmente esses 255 valores de cada cor; por exemplo: rgb(242, 237, 237), 242=F2, 237=ED, 237=ED, então fica #F2EDED.

Listas HTML

Para exibir uma listagem não ordenada:

<ul>
    <li>Primeiro item da lista</li>
    <li>Segundo item da lista:
        <ul>
            <li>Primeiro item da lista aninhada, 
            ou seja, tópicos de um tópico da lista</li>
            <li>Segundo item da lista aninhada</li>
        </ul>
    </li>
    <li>Terceiro item da lista</li>
</ul>

Para exibir uma listagem ordenada:

<ol>
    <li>Primeiro item da lista</li>
    <li>Segundo item da lista</li>
    <li>Terceiro item da lista</li>
    <li>Quarto item da lista</li>
</ol>

Podemos misturar esses dois tipos também.

Existe também o tipo de glossário, que usamos para listar termos e significados, que é chamada de lista de definição:

<dl>
    <dt>TERMO1</dt>
    <dd>
        Significado do termo1
    </dd>
    <dt>TERMO2</dt>
    <dd>
        Significado do termo2
    </dd>
</dl>

Espaçamento, Margem e Dimensões

Padding

Espaçamento: Para definir espaçamento interno em algum elemento, ou seja, o limite de sua borda ao conteúdo.

padding-top

padding-right

padding-bottom

padding-left

Essas propriedades aplicam uma distância entre o limite do elemento e seu conteúdo acima, direita, abaixo e esquerda. Podemos definir essas subpropriedades com um único comando que seja jogado para todos: 

p {
    padding: 10px;
}

Se passar dois valores, o primeiro será mandado para cima e para baixo, o segundo para esquerda e direita.

Se passar 3 valores, o primeiro será mandado para cima, o segundo para direita e esquerda, e o terceiro para baixo.

Se passar 4 valores, respectivamente: cima, direita, baixo, esquerda.

Margin

Margem: Parecido com a anterior, mas vai dar um espaço após o elemento em si, ou seja, além dele.

As subpropriedades são as mesmas e funcionam da mesma forma:

margin-top

margin-right

margin-bottom

margin-left

O margin permite que o navegador defina qual será a dimensão de padding ou margin conforme o espaço disponível na tela; usamos o valor "auto".

Para isso é só colocar auto no lugar do valor que você deseja substituir, exemplo:

p {
    margin: 0 auto;
}

Então pela regra, temos dois valores, o primeiro seria para cima e baixo, que eu coloquei 0, e o segundo para direita e esquerda, que eu coloquei auto.

Dimensões

Usamos o height (altura) e width (largura):

p {
    height: 300px;
    width: 300px;
}

Todos os parágrafos do nosso HTML ocuparão 300 pixels de altura e de largura.

Elementos estruturais

Quando não achamos a tag certa para um conteúdo, podemos usar div ou span, que são como coringas. E veremos que também é estilizável.

CSS: Seletores de ID e filho

Sabemos que é possível selecionar elementos no CSS usando o nome da tag, mas e se quiséssemos pegar um único parágrafo?

Seletor ID

Colocaremos um ID na tag HTML, e para pegar no CSS usamos esse ID, que deve ter um # antes.

Seletor hierárquico

Podemos ainda utilizar um seletor hierárquico que permite colocar estilos no filho de um elemento mãe. Para isso você coloca o ID na mãe, e na hora de selecionar no CSS, usa o ID + tag filha. 

<img src="logo-topo.png">
<div id="rodape">
    <p>Siga-nos nas redes sociais:</p>
    <img src="facebook-icon.png">
    <img src="instagram-icon.png">
</div>

Se eu quero apenas as imagens do facebook e instagram, devo colocar assim no CSS:

#rodape img {
    margin-right: 30px;
    vertical-align: middle;
    width: 94px;
}

Fluxo do documento e float

Perceba que o HTML por padrão deixa todo o conteúdo um embaixo do outro, como caixas empilhadas; para sair desse fluxo de documento, podemos usar o float.

.foto-familia {
    float: left; /* Faz a imagem flutuar para a esquerda */
    margin-right: 20px; /* Cria um espaço entre a imagem e o 
        texto ao lado */
    margin-bottom: 10px; /* Cria um espaço para o texto 
        não colar embaixo */
    width: 200px; /* Define um tamanho para a imagem */
}

Como o elemento com float sai do fluxo normal da página, os elementos que vêm muito depois dele (como um rodapé ou outra seção) podem tentar subir e se misturar de forma bagunçada. Para evitar que isso aconteça, nós usamos uma propriedade chamada clear no elemento que deve ficar obrigatoriamente abaixo de tudo, reiniciando o fluxo normal do HTML.

.elemento-de-baixo {
    clear: both; /* Impede que este elemento suba para o lado 
        de qualquer float */
}

Como colocar as faixinhas na direita e esquerda, centralizando o conteúdo

Isso não está no livro, ao menos não nas 50 primeiras páginas.

Primeiro colocamos a página inteira em uma div de nome qualquer; por padrão pessoal, eu escolho container.

Por que não usar o body? Porque ao querermos mover um bloco inteiro, ele necessariamente precisa ser um bloco, e body não é um bloco, ele é a tela toda; pode até se comportar como bloco, mas pode acabar tendo comportamentos inesperados.

Então limitamos o tamanho que o container central pode atingir com o width (largura), com um comando especial chamado:

max-width: valor;

Note que o max é justamente dizer: máximo da largura.

Coloque uma margin (margem) para centralizar; cima e baixo deixa em 0, pois não terão margem, e auto na esquerda e na direita, para o navegador dar a margem ideal.

Como o navegador dá a margem ideal?

"Bom, a tela do usuário tem 1920px de largura. O container só precisa de 960px. Sobraram 960px de espaço vazio. Como a margem das laterais está em 'auto', vou dividir esses 960px que sobraram igualmente: 480px para a esquerda e 480px para a direita."

Verá que tem umas bordinhas no topo e no final, além do texto estar muito colado no limite; usaremos o padding para driblar os dois problemas:

padding: 40px 20px; Pois, 40 no topo e no fim, e 20 nas laterais.

Por que padding e não margin?

Pois margin é a margem após o conteúdo, e padding é a margem do conteúdo em relação ao limite.

Acessibilidade

Para acessibilidade, tem padrão de:

p {
    line-height: 1.5; /* 1.5 sem aquele px significa,
         1.5x do valor atual, por exemplo: a fonte que você escolheu 
         tem 16px, então será 16px*1.5 */
    letter-spacing: 0.12em; /* tamanho do espaço entre cada letra */
    word-spacing: 0.16em;   /* tamanho do espaço entre cada palavra */
}

Mas é claro que isso vai depender da fonte e bom senso; verifique as diretrizes de acessibilidade.
