Processo de desenvolvimento de uma tela

Para desenvolver seu projeto WEB muitos fatores devem ser analisado, como por exemplo o publico alvo, para que assim seja escolhido a melhor abordagem para navegação em relação ao tom linguistico, visual, entre outro.

Assim a primeira etapa do desenvolvimento web fica a cargo da area de User Experience Design (UX) ou Interaction Design (IxD), onde temos pessoas formadas em comunicação

Essas pessoas vão definir varias caracteristicas humana do projeto, defindo o conteduo do Design de Interação, Experiencia de Usuario e Usabilidade. Assim, o mapa do site, esboço da visão, como as configurações.

Esse esboço é chamado de wireframes, ou seja.

WIREFRAMES

É o esboço da visão do projeto, baseado na User Experience Desing. E apartir dele que podemos construir nosso site, é depois dele que vem cor, fonte, etc.

//Imagem

Agora que teos o wireframe, o designe grafico pode colocar imagens, cores, tipos, fundos, bordas. Esse trabalho é chamado de layout, ou seja:

Layout

Imagens estaticas ja com o visual completo do site, basicamente oque fazemos no canva, a parte de programação entra depois do Layout feito.

//Imagem

Agora com o layout, o programador front-end vai usar tecnologias para o programa, como HTML, CSS, JAVASCRIPT, ADOBE FLASH, etc. Obs: para interativida usamos bastante javascript,mas falaremos disso em outro momento

HTML SEMANTICO

Para entender isso, veremos dois casos a seguir:

<body>
    <div>
        <!--Conteudo Cabeçalho-->
    </div>

    <div>
        <!--Conteudo Rodapé-->
    </div>
</body>

No caso acima temos uma div para o rodapé e uma para o cabeçalho, ao ler esse arquivo só descobrimos ter essa finalidade quando lemos o conteudo, talves quando lemos uma id/class espeficicando sua função, agora veja o caso a seguir, onde o html5 cria uma nova tag, uma só para rodapé, e outra para cabeçalho 

<body>
    <header>
        <!--Conteudo Cabeçalho-->
    </header>

    <footer>
        <!--Conteudo Rodapé-->
    </footer>
</body>

Agora, essa tag significa alguma coisa, não é algo generico, isso é chamado de semantica, ou seja:

SEMANTICA HTML

É o significado que a tag da ao conteudo, saindo do generico e representando de fato algo

ALem disso, robôs de busca como o Google não leem de forma visual, então ele vai ler o codigo html, e para entender oque deve ser oque. Semântica é uma importante técnica de SEO - Search Engine Optimization
