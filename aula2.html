HTML SEMANTICO

Para entender isso, veremos dois casos a seguir:

<body>
    <div>
        <!--Conteudo Cabeçalho-->
    </div>

    <div>
        <!--Conteudo Rodapé-->
    </div>

No caso acima temos uma div para o rodapé e uma para o cabeçalho, ao ler esse arquivo só descobrimos ter essa finalidade quando lemos o conteudo, talves quando lemos uma id/class espeficicando sua função, agora veja o caso a seguir, onde o html5 cria uma nova tag, uma só para rodapé, e outra para cabeçalho

    <header>
        <!--Conteudo Cabeçalho-->
    </header>

    <footer>
        <!--Conteudo Rodapé-->

</body>

Agora, essa tag significa alguma coisa, não é algo generico, isso é chamado de semantica, ou seja:

SEMANTICA HTML

É o significado que a tag da ao conteudo, saindo do generico e representando de fato algo

ALem disso, robôs de busca como o Google não leem de forma visual, então ele vai ler o codigo html, e para entender oque deve ser oque. Semântica é uma importante técnica de SEO - Search Engine Optimization

Pensando no HEADER

Vimos então que o cabeçalho sera representado pela tag <header> semanticamente, agora veremos o conteúdo dele

Não é uma regra, mas muitas vezes vemos links, como: sobre, contatos, etc.

Podemos usar uma lista <ul> com <li> e para transformar essas listas em link usamos a tag <a>

Assim esta bom, mas para melhorar essas lista <ul> podemos usar a tag <nav> para agrupar os links de navegação.

Novamente, nav visualmente não interfere, mas semanticamente, o navegador entende que oque estiver dentro dele é um agrupado de links para outros conteúdos.

Assim, o cabeçalho ficaria assim:

<header>
    <nav>
        <ul>
            <li>
                <a href="#">>Sobre</a>
            </li>
            <li>
                <a href="#">>Contato</a>
            </li>
        </ul>
    </nav>
</header>

O # significa que o link aponta para uma âncora dentro da mesma página. O que significa é que o link vai rolar a página até encontrar o elemento com o id correspondente. No entanto, para que isso funcione, é necessário ter um elemento com o id correspondente na página. Dessa forma:

<div id="sobre">Sobre</div>

Mas podemos colocar o link para um pagina externa, ao inves de colocar #, basta especificar a URL completa

No cabeçalho usamos muitas vezes logo, que geralemente é uma imagem, é comum colocar essa imagem dentro de um h1, para que o navegador associe o titulo, que geralemente é o nome da empresa, com a imagem do logo, assim o navegador entende que a imagem é o logo da empresa, e não apenas uma imagem qualquer.

Ficando assim:

<h1><img src="logo.png" alt="Logo da Empresa"></h1>

Block vs Inline

As tag que são exibidas em bloco ocupam toda a largura disponível, enquanto as tags inline ocupam apenas o espaço necessário, permitindo, naturlmente, que outras coisas fiquem ao seu lado.

Exemplo de tags em bloco: <div>, <h1>, <p>

Exemplo de tags inline: <span>, <a>, <img>

As tags inline não podem ter largura e altura definidas, enquanto as tags em bloco podem ter largura e altura definidas. As tags inline não podem ter margens e preenchimentos definidos, enquanto as tags em bloco podem ter margens e preenchimentos definidos. As tags inline não podem ter bordas definidas, enquanto as tags em bloco podem ter bordas definidas.

Existe uma propriedade chamada "Display" que permite definir a estrategia de exibição

Dessa forma, definimos o display:block; caso queiramos que o elemento se comporto como bloco. Ou display:inline; caso queiramos que o elemento se comporto como inline.

Position: Static, Relative, Absolute

A propriedade Position, determina o modo de posicionamento dos elementos na página.

Static

Valor padrão. O elemento permanece no seu lugar original, ou seja, de renderização da pagina

Se parssarmos valores de coordenadas, ele não vai se mover.

Relative

O elemento é posicionado em relação à sua posição original. Ou seja, passamos valores e ela se move.

Exemplo:

.logotipo{
    position: relative;
    top: 10px;
}

O elemento se move 10px para baixo, mas o espaço que ele ocupava originalmente ainda é reservado.

Absolute

O elemento é posicionado em relação ao seu elemento pai mais próximo que tenha uma posição definida diferente de static. E vai obedece as coordenas de acordo com o tamanho total do elemento pai

E se não tiver elemento diferente de static para ele ter como referencia?

Então a referencia vai ser a porção visual da pagina.

Observação: A referencia é em relação a posição original do elemento de referência.

Fixed

Sempre terá como referencia a porção visivel do documento, e mantem-se naquela posição até mesmo quando tem rolagem na tela.

Posicionamento com FLOAT e CLEAR

Lembra que falamos do float?

Relembrando: o float serve para fazer um bloco flutuar e ficar a esquerda ou direita de um outro elemento, e não abaixo dele.

O clear serve para ser aplicado nos elementos que vem depois do eemento que esta flutuando. Para que ele não fique flutuando tambem.

Ou seja, ele diz para, apartir daquele elemento, parar a flutuação.

Voce deve indicar que lado voce quer que pare de flutuar, com o left e right.

Dessa forma:

.conteudo{
    float: left;
}
.rodape{
    clear: left;
}

Decoração de Texto com CSS

O CSS permite que você decore o texto de várias maneiras.

Transformar Texto (text-transform)
Realiza transformações no texto, como converter para maiúsculas ou minúsculas.
  • uppercase: transforma todo o texto em maiúsculas.
  • lowercase: transforma todo o texto em minúsculas.
  • capitalize: transforma a primeira letra de cada palavra em maiúscula.
Decoração de texto
Permite aplicar estilos de decoração ao texto, como sublinhado, itálico e negrito.
  • underline: aplica um sublinhado ao texto.
  • overline: aplica uma linha acima do texto.
  • line-through: aplica uma linha atravessando o texto.
  • none: remove qualquer decoração de texto.
Espaçamento entre letras (letter-spacing)
Define o espaçamento entre as letras do texto.
  • normal: utiliza o espaçamento padrão entre as letras.
  • valor em px, em, rem, etc.: define um espaçamento específico entre as letras.
Espaçamento entre palavras (word-spacing)
Define o espaçamento entre as palavras do texto.
  • normal: utiliza o espaçamento padrão entre as palavras.
  • valor em px, em, rem, etc.: define um espaçamento específico entre as palavras.
Altura da linha (line-height)
Define a altura da linha do texto, ou seja, o espaçamento vertical entre as linhas.
  • normal: utiliza a altura de linha padrão.
  • valor em px, em, rem, etc.: define uma altura de linha específica.

Cascata e Herança no CSS

Algumas propriedades de elementos pais, quando alteradas, podem afetar os filhos.

Por exemplo, se alterarmos a cor do texto de um elemento pai, todos os elementos filhos herdarão essa cor, a menos que tenham uma cor específica definida.

As propriedades que não são aplicadas em cascatas são alequelas que afetam diretamente a caixa (box) do elemento.

Exemplos de propriedades que não são aplicadas em cascata incluem:

  • width
  • height
  • margin
  • padding
  • border

Por exemplo:

h1{
    padding-left: 40 px;
}
#pai{
    padding-left: 0;
}

Nesse exemplo o padding do h1 não sera sobrescrito pelo valor do elemento pai, ou seja, mantera o valor de 40 px

O valor inherit

Mas e se quisermos que um elemento herde o valor de uma propriedade de seu pai? Utilizamos o valor inherit.

Por exemplo:

img{
    width: inherit;
}

O valor inherit diz ao elemento para usar o mesmo valor presente no elemento pai.

Seletores de atributo do css3

Alem de tag, classe e id, temos outros seletores avançados.

Um dos mais versateis é o seletor de atributo. Por exemplo:

input[value] {
    color: blue;
}

O seletor [value] aplica o estilo a todos os elementos input que possuem o atributo value.

Podemos também usar seletores de atributo com valores específicos, como:

input[type="text"] {
    border-radius: 4px;
}

O seletor [type="text"] aplica o estilo apenas aos elementos input que possuem o atributo type com o valor "text".

É possivel tambem usar operadores com determinada condição. Como por exemplo:

div[class|="menu"] {
    border-radius: 4px;
}

O seletor [class|="menu"] aplica o estilo apenas aos elementos div que possuem a classe que começa com "menu". Podendo ser qualquer outra coisa apos o hifen, como menu-principal, menu-teste, etc.

Podemos buscr por uma palavra-chave em particular:

input[value≃"problema"] {
    background-color: lightblue;
}

O seletor [value≃"problema"] aplica o estilo apenas aos elementos input que possuem o atributo value contendo a palavra "problema".

Podemos buscar por um valor que termina com uma palavra específica:

input[value$="problema"] {
    background-color: lightblue;
}

O seletor [value$="problema"] aplica o estilo apenas aos elementos input que possuem o atributo value terminando com a palavra "problema".

Substituição por imagem

Image Replacement ou substituição por imagem, é bem facil de entender. Mas primeiro lembraremos de algumas coisas

Quando colocamos uma imagem, mesmo que um link, usamos a tag <img>, e lembra-se que temos o alt no img, para que dessa forma, se a imagem falhar por algum motivo, ou por acessibilidade (extensões para deficiente visual que precisa que o navegador leia para eles, por exemplo), o alt vai dizer oque seria a tal imagem

Mas supomos que vamos fazer uma imagem com css, não colocar a imagem em si, por exemplo, podemos recriar o logo do youtube facilmente com css, ao inves de colocar uma imagem, uma vez que fizemos essa imagem com o css, é fato que não temo mais o alt, pois o alt é da tag <img>, então, para isso fazzer uma "malabarismo" para para criar uma especie de alt, ou seja, uma descrição que não esta visualemnte ali, mas que o navegador vai ler, e que vai ser lido por extensões de acessibilidade, como o leitor de tela.

Então a substituição por imagem é o truque completo, a criação da imagem por css ao inves de uma img, e essa tecnica de fazer esse alt especial

Como que é feito esse alt forçado?

Vejamos um exemplo:

.logo{
    width: 200px;
    height: 100px;
    background-image: url('logo.png');
    text-indent: -9999px;   
}

Você reparou que existe uma URL ali? Alguém poderia pensar que, por ter uma URL apontando para um arquivo, isso não foi feito por CSS. Bem, o CSS pode não ter desenhado a imagem do zero, mas é ele quem carrega e exibe esse arquivo na tela, e não o HTML. Isso já entra no nosso critério de substituir a tag <img> pelo CSS.

O processo funciona assim:

Primeiro, o elemento HTML recebe um texto real (que funcionará como o nosso 'alt forçado'). Ajustamos o tamanho desse elemento no CSS para ficar igual ao tamanho da imagem.

Segundo, fazemos a imagem ser o background (fundo) desse elemento.

Terceiro, usamos o text-indent negativo para chutar a escrita para bem longe, jogando o texto para fora dos limites visíveis da tela. Dessa forma, o texto continua existindo no código para os leitores de tela, mas os nossos olhos só enxergam a imagem de fundo que ficou no lugar dele.

4.13 Estilização e Posicionamento do Rodapé

O rodapé tem um comportamento único no nosso layout: ele ocupa 100% da largura da tela para que o seu fundo preto estampado se repita até os cantos. Porém, o conteúdo dentro dele precisa ficar alinhado e limitado aos mesmos 940px do restante do site. Para resolver isso, usamos a nossa velha conhecida classe .container dentro dele.

Vamos entender como resolver o posicionamento e aplicar o truque que acabamos de aprender!

1. O Problema do Posicionamento (O Rodapé Subiu!)

Ao colocar o rodapé no HTML, você vai reparar que ele vai subir na página e invadir o espaço dos outros elementos, em vez de ficar lá embaixo. Isso acontece porque os elementos anteriores (os painéis de destaque) estão usando float e saíram do fluxo normal de renderização da página. Para corrigir isso e forçar o rodapé a ir para o seu devido lugar (embaixo de tudo), usamos a propriedade clear: both nele.

2. Alinhando os Ícones das Redes Sociais

Queremos que a lista de ícones seja jogada para o lado direito do rodapé. Para fazer isso, usamos o posicionamento absoluto na lista. Mas lembre-se da regra: para o posicionamento absoluto funcionar em relação ao rodapé, o container do rodapé precisa estar posicionado como position: relative.

Além disso, por padrão, os itens da lista (<li>) ficam um embaixo do outro. Para fazê-los ficarem lado a lado, aplicamos float: left em cada um deles.

3. Aplicando a Substituição por Imagem (Image Replacement)

Agora vem o truque completo que estudamos! Os links das redes sociais no HTML terão apenas textos normais (como "Facebook", "Twitter"). No CSS, nós vamos aplicar a técnica de Image Replacement para chutar esse texto para longe com o text-indent negativo e exibir apenas o ícone como background-image.

Para o CSS saber exatamente qual imagem colocar em qual link sem que a gente precise criar uma classe para cada um, usamos um seletor de atributo do CSS3. Ele varre o link e descobre a imagem certa com base no endereço do site!

Vejamos como fica esse trecho no código:

/* O seletor abaixo procura qualquer link que tenha "facebook.com" no href */
.social a[href*="facebook.com"] {
    width: 85px;  /* Tamanho exato da imagem */
    height: 55px; /* Tamanho exato da imagem */
    background-image: url('../img/facebook.png'); /* Imagem via CSS */
    text-indent: -9999px; /* Chuta o texto "Facebook" para fora da tela */
}

Com essa combinação de técnicas, o CSS cuida de todo o carregamento visual e do alinhamento, enquanto o nosso HTML continua limpo, organizado e 100% acessível para os leitores de tela.
